# Полная документация проекта ImagoRef

## Введение

**ImagoRef** — это десктопное кроссплатформенное приложение для создания, редактирования и управления мудбордами (досками референсов). Программа предоставляет пользователю бесконечный холст для работы с изображениями, поддерживает локальное сохранение в собственный формат файлов `.iref`, работает в оффлайн-режиме благодаря локальной базе данных и поддерживает облачную синхронизацию в реальном времени. Ключевой особенностью является встроенный ИИ для увеличения разрешения (апскейлинга) изображений.

---

## 1. Стек технологий и используемые библиотеки

Проект построен на современном стеке технологий C++ и Qt:
- **Язык**: C++17
- **Фреймворк**: **Qt 6** — основной фреймворк для разработки.
  - *Qt Core*: базовые классы (строки, таймеры, потоки).
  - *Qt Gui / Qt Quick / Qt Qml*: отрисовка графики, декларативный UI, интеграция с C++.
  - *Qt QuickControls2*: готовые компоненты интерфейса (кнопки, слайдеры).
  - *Qt Svg*: работа с векторной графикой (иконки).
  - *Qt Network*: HTTP-запросы к API.
  - *Qt WebSockets*: двунаправленная связь с сервером для синхронизации холста в реальном времени.
  - *Qt Sql*: работа с локальной базой данных (SQLite).
- **Библиотеки сжатия**: 
  - **QuaZip** (v1.4) — C++ обертка над zlib для работы с ZIP-архивами. Используется для сохранения и чтения файлов формата `.iref`.
  - **zlib** — стандартная библиотека сжатия данных.
- **ИИ и Нейросети**: 
  - **ncnn** (Tencent) — высокопроизводительный фреймворк для инференса нейросетей. Используется для локального апскейлинга картинок прямо на устройстве пользователя.
- **Аппаратное ускорение**:
  - **OpenGL** — используется Qt для аппаратного ускорения отрисовки графики и бесконечного холста.
- **Система сборки**: **CMake** (v3.21.1+).

## 2. Архитектура приложения

Приложение построено по паттерну **MVC/MVVM** с четким разделением бизнес-логики (написанной на C++) и представления (описанного в QML). Общение между слоями происходит через систему сигналов и слотов, а также через свойства `Q_PROPERTY`.

### 2.1. Контроллеры (Controllers)
Контроллеры отвечают за бизнес-логику и координацию действий. Расположены в `src/controllers/`.

1. **`BoardController`**
   - *Файл*: `BoardController.h / .cpp`
   - *Описание*: Главный фасадный контроллер. Является единой точкой входа для QML-интерфейса. Хранит ссылки на все остальные контроллеры и предоставляет их свойства (`storageController`, `toolController` и т.д.). Управляет глобальными параметрами холста: размер сетки, координаты камеры (`cameraX`, `cameraY`) и масштаб (`cameraZoom`), а также стэком отмены действий (`QUndoStack`).
   
   *Пример проброса свойства в QML (C++):*
   ```cpp
   // Свойство gridSize будет доступно напрямую в QML
   Q_PROPERTY(int gridSize READ getGridSize WRITE setGridSize NOTIFY gridSizeChanged)
   ```
   
2. **`StorageController`**
   - *Файл*: `StorageController.h / .cpp`
   - *Описание*: Отвечает за хранение данных. Инициализирует и управляет локальной базой данных SQLite. Обеспечивает сохранение/загрузку досок в формат `.iref` (упаковка JSON и картинок в ZIP-архив через QuaZip). Реализует механизм Offline-First.

3. **`NetworkController`**
   - *Файл*: `NetworkController.h / .cpp`
   - *Описание*: Отвечает за связь с бэкендом. Управляет REST API запросами и WebSocket-соединениями (`QWebSocket`) для коллаборативной работы. 

4. **`SelectionController`**
   - *Файл*: `SelectionController.h / .cpp`
   - *Описание*: Логика выделения объектов. Позволяет выделять одно изображение или несколько (Shift+Click, выделение рамкой), сбрасывать выделение.

5. **`ClipboardController`**
   - *Файл*: `ClipboardController.h / .cpp`
   - *Описание*: Отвечает за буфер обмена. Позволяет копировать картинки (локально или из ОС) и вставлять их на доску.

6. **`ToolController`**
   - *Файл*: `ToolController.h / .cpp`
   - *Описание*: Бизнес-логика инструментов редактора. Включает удаление, привязку к сетке, вращение, обрезку (Crop), добавление подписей, и работу пипетки (Eyedropper).

7. **`UpscaleController`**
   - *Файл*: `UpscaleController.h / .cpp`
   - *Описание*: Отвечает за ИИ-увеличение качества картинок. Запускает тяжелые вычисления в фоновом потоке через `UpscaleWorker`, передавая данные в библиотеку `ncnn`.

8. **`AuthController`**
   - *Файл*: `AuthController.h / .cpp`
   - *Описание*: Синглтон-контроллер авторизации. Взаимодействует с REST API сервера для логина и регистрации.

9. **`StackController`**
   - *Файл*: `StackController.h / .cpp`
   - *Описание*: Набор классов команд (Command Pattern), наследуемых от `QUndoCommand`. Реализует логику действий для стека Undo/Redo.

### 2.2. Менеджеры (Managers)
Менеджеры предоставляют глобальные сервисы. Расположены в `src/managers/`.

1. **`BoardsManager`**: Запрашивает у `StorageController` историю открытых досок для стартового меню.
2. **`SettingsManager`**: Загружает, хранит и применяет настройки пользователя.
3. **`ThemesManager`**: Читает файлы тем (Dark/Light) и применяет их к интерфейсу приложения.
4. **`ModelsManager`**: Загрузка и валидация весов нейросетевых моделей для `ncnn`.
5. **`CacheManager`**: Управляет дисковым кэшем изображений. Изображения сохраняются на диск с именем в виде SHA-256 хэша от их байт.

### 2.3. Модели данных (Models)
Расположены в `src/models/`.

1. **`ImagoImageData`** (Структура в `ImageModel.h`)
   - Хранит свойства одной картинки: `id`, `source`, `x`, `y`, `width`, `height`, `rotation`, `zValue`, `opacity`, `imageHash` и др.
2. **`ImageModel`** 
   - Наследуется от `QAbstractListModel`. Хранит список объектов `ImagoImageData`.
3. **`ImageProvider`**
   - Специальный провайдер (`QQuickImageProvider`), позволяющий QML асинхронно запрашивать картинки из кэша.

## 3. Пользовательский интерфейс (QML)

Все визуальные компоненты написаны на декларативном языке QML (директория `src/qml/`).

### Основные окна (`src/qml/windows/`)
- **`WelcomeWindow.qml`**: Стартовый экран (Auth + список досок).
- **`MainWindow.qml`**: Главный экран редактора.
- **`SettingsWindow.qml`**: Диалоговое окно настроек.

### Основные компоненты (`src/qml/components/`)
- **`CanvasView.qml`**: Реализация бесконечного холста. Обрабатывает панорамирование (pan) и масштабирование (zoom).
- **`ImageItem.qml`**: Отрисовывает одну картинку на холсте, применяя к ней трансформации из `ImageModel`.
  
  *Пример привязки данных модели к компоненту QML:*
  ```qml
  Image {
      x: model.x
      y: model.y
      width: model.width
      height: model.height
      rotation: model.rotation
      opacity: model.opacity
      source: "image://imagoref/" + model.imageHash
  }
  ```

### Оверлеи и Инструменты
- **`FloatingToolbar.qml`**: Плавающая основная панель инструментов.
- **`ResizeOverlay.qml`**: Рамка и угловые маркеры для изменения размера.
- **`CropOverlay.qml`**: Маска для кадрирования.
- **`EyedropperOverlay.qml`**: Инструмент пипетки.
- **`ColorPalettePanel.qml`**: Панель вывода сохраненных цветов.

## 4. Файловая система и форматы (Offline-First)

Приложение спроектировано для работы без интернета (Offline-first) и локального хранения.

### 4.1. Локальная база данных (SQLite)
База располагается в папке AppData (`imago_local.db`).

*Пример структуры таблицы `items`:*
```sql
CREATE TABLE IF NOT EXISTS items (
    id TEXT PRIMARY KEY,
    board_id TEXT,
    type TEXT,
    x REAL,
    y REAL,
    width REAL,
    height REAL,
    z_index INTEGER,
    payload TEXT, -- Вся остальная информация (rotation, crop) в виде JSON
    updated_at INTEGER,
    is_dirty INTEGER DEFAULT 1,
    is_deleted INTEGER DEFAULT 0
);
```
Флаги `is_dirty` и `is_deleted` отслеживают изменения для оффлайн-синхронизации.

### 4.2. Формат файлов `.iref`
Для ручного экспорта или переноса досок используется формат `.iref`. Это ZIP-архив, создаваемый через библиотеку `QuaZip`.
Архив содержит:
1. **Папку `images/`** — в нее копируются оригинальные картинки (в формате `.png`), названные по их ID (например, `images/abc-123.png`).
2. **Файл `data.json`** — метаданные.

*Пример файла `data.json`:*
```json
{
  "version": "1.0",
  "canvas": {
    "cameraX": 150.5,
    "cameraY": -20.0,
    "cameraZoom": 0.8,
    "gridSize": 25
  },
  "items": [
    {
      "id": "abc-123",
      "imagePath": "images/abc-123.png",
      "pos_x": 100.0,
      "pos_y": 100.0,
      "width": 500,
      "height": 500,
      "rotation": 15.5,
      "zValue": 1
    }
  ]
}
```

## 5. Как работают ключевые механизмы программы

### 5.1. Жизненный цикл добавления картинки
1. Пользователь нажимает "Вставить" или перетаскивает картинку на холст.
2. `ClipboardController` считывает байты картинки, считает SHA-256 хэш и отдает в `CacheManager`.
3. `CacheManager` сохраняет файл на диск как `<hash>.png`.
4. `ClipboardController` создает `AddImageCommand` и отправляет в `QUndoStack`.
5. Внутри `AddImageCommand::redo()`: `ImagoImageData` добавляется в `ImageModel`. Модель генерирует сигнал `dataChanged`.
6. QML `Repeater` ловит сигнал и создает новый `ImageItem.qml`.
7. Данные записываются в БД (таблица `items`) с флагом `is_dirty = 1`.

### 5.2. Undo/Redo и Трансформации
1. QML шлет координаты трансформации в контроллер (например, `BoardController::beginMove()`).
2. При завершении отправляются конечные координаты (`BoardController::endMove()`).
3. Формируется команда для стека отмены (например `MoveImageCommand`).

*Пример логики команды (C++):*
```cpp
class MoveImageCommand : public QUndoCommand {
public:
    void undo() override {
        // Логика отмены (Возврат к старой позиции)
        m_model->setPosition(m_index, m_oldPos.x(), m_oldPos.y());
    }
    void redo() override {
        // Логика выполнения (Переход к новой позиции)
        m_model->setPosition(m_index, m_newPos.x(), m_newPos.y());
    }
private:
    ImagoImageModel *m_model;
    int m_index;
    QPointF m_oldPos, m_newPos;
};
```

### 5.3. ИИ Увеличение (Upscale)
1. В `QThreadPool` добавляется задача `UpscaleWorker`. Интерфейс не зависает.
2. В рабочем потоке загружаются тензоры в `ncnn`, проводится инференс (увеличение разрешения).
3. По завершении отправляется сигнал `finished(index, QImage)` в GUI поток.
4. Главный поток обновляет картинку в модели.

### 5.4. Синхронизация (Network Sync)
При появлении сети `NetworkController` запрашивает у БД JSON со всеми `updated_items` (`is_dirty = 1`) и `deleted_items` (`is_deleted = 1`). Он отправляет эти данные по WebSocket. При успехе вызывается `markAsSynced()`, снимающий флаги и физически удаляющий "мягко" удаленные элементы из базы.

## 6. Сборка и деплой (CI/CD)

Процесс сборки автоматизирован через CMake (файлы `CMakeLists.txt`, `Info.plist.in`, `resources.rc`):
- **macOS**: Скрипт использует `macdeployqt` для сбора библиотек, `codesign` для подписи и `pkgbuild / productbuild` для упаковки приложения в нативный инсталлятор `.pkg`.
- **Windows**: Собирает `.exe`. Использует генератор `NSIS` для создания инсталлятора, который автоматически прописывает приложение в реестре (ассоциация с файлами `.iref`).
- **Linux**: Копирует файл `Info.desktop.in` в `/share/applications` для интеграции с DE и собирает бинарный файл (или AppImage).
